[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the flow diagram to understand how it works under the hood. 
(pause)
First, the user message enters the START node and immediately goes to our security checkpoint function node, which scans for PII and prompt injection attacks.
If a prompt injection is detected, it immediately routes to the security event node to block the request.
Otherwise, it routes to the document ingestion node. Here, if no document content is present in the session state, the agent utilizes a human-in-the-loop pause to ask the user to provide it.
Once the document is registered in state, the message is sent to our orchestrator agent.
The orchestrator agent coordinates the user's intent and uses AgentTool to delegate tasks to either the summarizer agent, for custom length summaries, or the qa expert agent, for specific question-answering.
Both agents can also utilize our local MCP server tools to read files, fetch webpage contents, compute metadata, or save the output.

[3:00 — SECURITY & MCP]
Our security checkpoint runs PII scrubbing, keyword injection detection, and confidentiality checks, outputting a machine-readable JSON audit log for compliance.
Meanwhile, our local Model Context Protocol server exposes tools to read workspace files, fetch web articles, compile text statistics, and write summaries back to disk.
